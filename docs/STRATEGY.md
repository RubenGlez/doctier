# `doctier` strategy and positioning

> Living document. Fixes `doctier`'s **value proposition** and **roadmap decisions**.
> It draws on market research (July 2026) on how the ecosystem manages the
> context documents generated by AI agents. The space moves in a matter of months:
> **revisit this map periodically** (§6). For the technical design, see [`DESIGN.md`](DESIGN.md).

## 1. Value proposition

**`doctier` is the _tier_ layer that AGENTS.md lacks: visibility × lifetime for
generated documents, on top of git, worktree-native, encrypted with age and harness-agnostic.**

Git models a single axis with two states (tracked / ignored). `doctier` adds the **two axes
that agent workflows really need** and that no current convention has:

- **Visibility** — `public` (plaintext) · `private` (encrypted with age, reusing SSH keys).
- **Lifetime** — `durable` (forever) · `ephemeral` (finite lifetime, automatic collection).

With **fail-closed** (impossible to publish private content in cleartext by accident) and **worktrees as a
first-class case** (parallel agents, no seeding hooks).

## 2. The problem and where the conversation is heading

Agent workflows generate PRDs, research, decision maps, plans and memory. Today
they end up in git "because there's nowhere else to put them", or gitignored wholesale (nothing is backed up or
travels). The pain is well articulated in the threads by **[@mattpocockuk](https://x.com/mattpocockuk/status/2069698109492343101)**
and **[@dexhorthy](https://x.com/dexhorthy/status/2069143768901791934)** (June 2026).

**Key research finding (and `doctier`'s underlying decision):** the thread asked for
storage _outside_ git with linear history. But **the serious ecosystem has already converged on the
opposite — git as the substrate** — and that's where `doctier` plays:

- **Letta Context Repositories** — git-backed agent memory, cloned to the filesystem,
  with each subagent in an **isolated worktree** and git-based merge. ([letta.com](https://www.letta.com/blog/context-repositories/))
- **Spec Kitty** — specs/plans/tasks versioned in the repo, parallel agents in
  **worktrees**. ([github](https://github.com/Priivacy-ai/spec-kitty))
- **Martin Fowler's taxonomy** (spec-driven development) — tools already segment by
  **lifecycle**: spec-first (throwaway), spec-anchored (persists), spec-as-source. The
  "lifetime" axis is real and already divides the market. ([martinfowler.com](https://martinfowler.com/articles/exploring-gen-ai/sdd-3-tools.html))

In other words: **the "on top of git, not outside" bet is aligned with the emerging consensus.** The thread
describes the pain; the ecosystem has already decided the substrate.

Two nuances that validate `doctier`'s axes:

- **The plan is ephemeral by design.** Pocock's workflow externalizes the plan to PRD/issue files
  _so they can be thrown away_, treats the PRD as a "reference marker, not compiler input", and
  assumes _doc rot_ (a later agent reads an old plan, believes it authoritative and produces garbage). Today
  it's managed by hand (closing issues, `/clear`). `doctier`'s **ephemeral + GC** axis automates that
  human decision to discard.
- **Discovery conventions won, but they're evergreen and have no access control.**
  AGENTS.md is the cross-tool standard (20k+ repos; Codex, Claude, Cursor, Copilot, Gemini; now
  under the Linux Foundation) with hierarchical per-path precedence; `CLAUDE.md` auto-loads and
  merges. But **no convention has a lifetime or visibility axis**: the only control is
  gitignore, and "they're not meant for secrets". This is the central gap. ([agents.md](https://agents.md), [code.claude.com](https://code.claude.com/docs/en/memory))

## 3. Competitive map — each owns one axis, no one owns the crossproduct

| Player | Axis it owns | What it lacks vs `doctier` |
|---|---|---|
| **Letta** | git-backed memory + worktrees | no visibility tiers, no lifetime, no encryption |
| **Spec Kitty** | repo-native specs + worktrees | cleartext, specs only |
| **Paper control-plane** ([arxiv jun-2026](https://arxiv.org/html/2606.26924v1)) | deterministic permission tiers, fail-closed | gates _tools_, not _docs_; no storage/lifecycle |
| **SOPS / age** ([getsops.io](https://getsops.io/)) | encryption on top of git | no lifecycle, no discovery, no injection |
| **AGENTS.md / CLAUDE.md** | discovery + auto-injection | evergreen only; gitignore as the sole access control |

**No tool combines visibility × lifetime × worktree-native × encryption × harness-agnostic.** That
crossproduct is `doctier`'s gap, confirmed by absence of counter-evidence across 22 sources.

## 4. Defensible differentiation

Three capabilities that come out of the axis crossproduct and that the report marks as open ecosystem
gaps:

1. **Decrypt-into-context (the smudge filter).** Agents need _plaintext at read
   time_, but the repo must store ciphertext. `doctier` decrypts on checkout: the agent reads
   cleartext locally, git stores encrypted. The report flags this pattern as an **open question**
   for the ecosystem; `doctier` already solves it.
2. **Worktree-native via tracked-encrypted.** Since the private doc is _tracked_ (encrypted), it travels
   natively via git to each worktree **without `.worktreeinclude` or seeding hooks**. The report
   confirms that gitignored content has to be duplicated per-worktree; `doctier` avoids it.
3. **Ephemeral + deterministic GC.** Automates the discard that's manual today, with
   `pr-merge` / `worktree` / `ttl` triggers and a fail-closed `check`.

## 5. Roadmap implications

Priorities derived from the analysis (order = strategic value):

- **P0 — Harden the crossproduct (already in prototype).** The differentiator is the _combination_ of axes, not
  each axis separately. Harden the manifest, clean/smudge filters, `check` and `gc` until the
  crossproduct is solid end-to-end before expanding surface area.
- **P1 — Bridge to AGENTS.md (highest-value move).** Automatic discoverability is the only
  one of the thread's needs that `doctier` does **not** cover, and AGENTS.md already owns it. **Don't compete:
  integrate.** `doctier` should emit _tier-aware_ pointers into AGENTS.md/CLAUDE.md (e.g. generate
  a block listing the docs visible to the agent depending on the work state). This closes the
  gap and turns AGENTS.md from a competitor into a distribution channel.
- **Per-host `gc` recipes (CI)** for the `pr-merge` trigger, which is intrinsically
  host-dependent (see [`DESIGN.md`](DESIGN.md) §7.1, §14).

**Non-goals to keep** (the report confirms they sustain the defensible niche):

- Don't be a document manager, wiki or **collaboration/comments** tool. No one serious
  solves it this way; adding it dilutes the focus.
- Don't compete with spec-driven tools (Spec Kit, Kiro, Tessl) or memory tools (Letta): `doctier`
  is the **cross-cutting layer** none of them has, not a substitute for any.

## 6. Main risk and window

**Threat:** if AGENTS.md (now under the Linux Foundation) adds a _lifecycle_ or _permission-
declaration_ extension, it absorbs `doctier`'s two axes from the convention side, not the
tooling side. The window is about **closing and making the crossproduct known before that happens**, and about
the bridge to AGENTS.md (P1) positioning `doctier` as the natural implementer of those axes if the
convention standardizes them.

**Time-sensitivity:** the space moves in months (paper jun-2026, `.worktreeinclude` jul-2026,
AGENTS.md → Linux Foundation). This map is a snapshot; re-verify competitors and conventions every
few months.
